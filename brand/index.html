01 — Philosophy / 理念

我們不是用 AI 把程式碼寫得更快，
而是用 AI 做出一個人做不出的東西。

We don't use AI to write code faster. We use AI to build what one person couldn't build alone.

人
Human — 洞察

願景、品味，以及真正重要的決定。方向由人來定。

技
AI — 執行

迭代、規模，以及所有可以被放大的事。力量交給 AI。

實
Together — 成品

一個開發者，現在能出貨的東西。這不是取代人，是解放人。

程式碼，即證明。 The code is the proof. Clone it. Read it. Run it. Break it.

05 — Naming System / 命名系統

每個名字，都是一個人類知覺的動作。

RikaiDev 最強的識別資產，是它的命名法。每個產品都取一個日文和語、以漢字承載多層詩意，並讓「詞的本義」直接對應工具在做的事。在 agent 遍地、模型會被換掉的時代，名字描述的永遠是人的動作——讀、看、守望——而非當下的技術。技術會過時，理解不會。

一
和語為名

取一個真實的日文詞，以小寫羅馬字作為產品標記。

二
漢字為體

名字皆有漢字，且往往一字多義（読み＝讀／詠／黄泉）。

三
意象即功能

詞的本義直接對應工具在做的事（目線→視角）。

四
以人為主詞

名字描述人的知覺與關懷；AI 是執行，名字是人。

五
克制字形

標記一律小寫無襯線；漢字只在印章與 emblem 出現。

cortex

早期以英文命名的先行專案，今已封存。它提醒我們：命名法則於此後才確立，往後一律和名。

命名配方 / How to name a new product
1 → 這個工具幫人做的，是哪一個「知覺或關懷的動作」？（讀、看、守望、收成…）
2 → 找出承載那個動作的日文和語，取其漢字。
3 → 確認漢字有第二層意象可延伸（越詩意越好）。
4 → 以小寫羅馬字定為產品標記，漢字入朱印。
✓ 名字說的是人；工具，只是讓那個動作成真。
06 — Graphic Language / 圖形語言

細線、方印、與一筆的餘白。

線圖標

1.5px 等粗細墨線、圓角 2px、方形收邊。單色，不填色，不加陰影。

方印母題

朱色方印可作為重複母題，微幅隨機旋轉（±3°）帶來手作的不完美。

細線網格

以髮絲級細線分割版面，取代填色卡片。留白為主，線為輔。

07 — Voice & Tone / 語調

誠實、克制，像一位好教練。

誠實的錯誤
Honest error
寧可誠實地失敗，也不偽造成功。
EN　Silence is honest; a fabricated result is not.
JA　正直な沈黙、偽りの成功なし。
教練，而非取代
Coach, not replacement
教你為什麼，而不只是怎麼做。
EN　Learn why, not what.
JA　答えより、理由を。
在公開場合建造
Building in public
程式碼即證明，不是投影片，不是 Twitter thread。
EN　The code is the proof. Not slides.
JA　コードが証明。
克制
Restraint
少即是多；每一個「是」，背後有一千個「不」。
EN　Less, but better.
JA　引き算の美学。
